FastAPI User Management API

Overview

This project is a Simple User Management REST API built using FastAPI.
The primary goal of this project is to learn and demonstrate FastAPI fundamentals such as routing, HTTP methods, request/response handling, middleware, validation, and API documentation.

The application manages users in memory (no database), as per assignment requirements.



Objectives

Learn FastAPI basics through hands-on implementation

Understand REST API design

Use Pydantic for request validation

Implement middleware

Work with query parameters and path parameters

Generate automatic API documentation using Swagger UI and ReDoc

Refactor code to follow a modular structure for better maintainability




Features

Home endpoint to verify API status

Create users using request body

Fetch all users with optional query parameter (limit)

Fetch user details using path parameter (user_id)

Delete user by ID

Custom middleware that adds response headers

Input validation using Pydantic

Auto-generated API documentation (Swagger UI & ReDoc)



API Endpoints
Home

GET /
Returns API status

{
  "message": "FastAPI is running"
}



Create User

POST /users

Request Body:

{
  "name": "John",
  "age": 22
}


Response:

{
  "id": 1,
  "name": "John",
  "age": 22
}


Get All Users

GET /users


Optional Query Parameter:

/users?limit=2


Get User by ID

GET /users/{user_id}


Delete User

DELETE /users/{user_id}


Project Structure 

app/
├── main.py        # Application entry point
├── routes.py      # API routes
├── models.py      # Pydantic schemas and validation
├── middleware.py  # Custom middleware
├── storage.py     # In-memory user storage


Tech Stack

1 Python

2 FastAPI

3 Uvicorn

4 Pydantic


Setup & Installation

1. Install Dependencies
pip install fastapi uvicorn

2. Run the Application
uvicorn main:app --reload

3. Open API Documentation


Swagger UI:
http://127.0.0.1:8000/docs

ReDoc:
http://127.0.0.1:8000/redoc

Notes

Data is stored in memory, so it resets on server restart

No database is used (intentional for learning)

Authentication and authorization are not implemented

This project focuses on understanding FastAPI fundamentals
